# Preparação para Certificações Camunda

Este repositório organiza o conteúdo, notas e exercícios relacionados à preparação para as certificações **Camunda Certified Associate (CCA)** e **Camunda Certified Professional (CCP)**. O objetivo é centralizar estudos, práticas e referências para ambos os exames.

---

## Objetivo

- Consolidar conhecimentos em BPMN, DMN e CMMN.
- Preparar-se para a certificação Camunda Associate.
- Preparar-se para a certificação Camunda Professional.
- Criar um repositório de estudo estruturado e consultável.

---

```

├── associate/

│   ├── dmn/

├── professional/
│   ├── casos-de-uso/
│   ├── advanced-bpmn/
│   └── notas.md
├── simulados/
│   └── quizzes.md
├── referencias/
│   └── links.md
└── README.md

````

- **associate/**: Conteúdo e exercícios para a certificação Associate.  
- **professional/**: Conteúdo avançado e casos de uso para a certificação Professional.  
- **simulados/**: Perguntas de revisão e quizzes práticos.  
- **referencias/**: Documentação oficial, links e materiais de apoio.

## Conteúdo

### BPMN
- Modelagem de processos.
- Eventos, gateways e tarefas.
- Subprocessos e Call Activities.
- Pools e Lanes.

### DMN
- Tabelas de decisão.
- Regras e prioridades.
- Expressões FEEL.
- Integração com BPMN.

### Camunda Platform
- Instalação e configuração.
- Engine API e REST.
- Camunda Modeler.
- Execução e monitoramento de processos.

### Estudos de Caso
- Processos de negócios reais.
- Automação de tarefas com Camunda.
- Análise de decisões complexas.

## Como utilizar este repositório

1. Clone o repositório:
```bash
git clone https://github.com/seu-usuario/camunda-certificacao.git
````

2. Navegue para o diretório do nível de estudo desejado:

```bash
cd associate
# ou
cd professional
```

3. Abra os arquivos `notas.md` e os exemplos no Camunda Modeler para prática.
4. Utilize a pasta `simulados/` para revisão periódica.

## Contribuição

Para contribuir com exemplos ou notas adicionais:

1. Faça um fork do repositório.
2. Crie uma branch: `git checkout -b minha-contribuicao`.
3. Adicione suas alterações e faça commit: `git commit -am 'Adicionando estudo'`.
4. Faça push para a branch: `git push origin minha-contribuicao`.
5. Abra um Pull Request para análise.
